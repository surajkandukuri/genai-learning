AI Concepts and Terminology

This document summarizes key AI and Machine Learning concepts in a structured format, providing clarity and real-world application.

Basic Terminology

  • AI → The broad field that enables machines to perform tasks requiring intelligence.
  • ML (Machine Learning) → A subset of AI where models learn from data instead of following fixed rules.
  • Deep Learning → A subset of ML that uses neural networks with multiple layers to learn complex patterns.
  • Foundation Models → Large-scale pre-trained models that serve as the base for many AI applications.
  • Generative AI (GenAI) → A form of AI that creates/generates content (text, images, music, or code).
  • Data Science → Using ML, statistics, and programming to find patterns in data.
  • RAG (Retrieval-Augmented Generation) → AI technique that fetches external data before generating responses.
  • State Space → The set of all possible situations a system can be in.
  • Transitions → The process of moving from one state to another.
  • Heuristics → AI’s version of filtering conditions to optimize decision-making.
  • Bayes Algorithm → AI updates probability dynamically as it receives new data.

Example: Generative AI vs Foundation Models

Think of a Foundation Model as a general-purpose brain, while Generative AI is a content creator that uses this brain to generate new content.

RAG (Retrieval-Augmented Generation)

Without RAG: AI can only respond based on old, pre-trained data.

With RAG: AI retrieves fresh information from databases before responding.

Final Thoughts

Understanding these AI concepts will help in navigating the evolving AI landscape and leveraging AI models effectively in enterprise environments.
